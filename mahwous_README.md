## 🚀 البدء السريع

### 1. إنشاء Repository

```bash
git clone https://github.com/YOUR_USERNAME/mahwous-engine.git
cd mahwous-engine
```

### 2. إضافة Secret المطلوب

اذهب إلى: **Settings → Secrets and variables → Actions → New repository secret**

| اسم الـ Secret | القيمة |
|:---|:---|
| `ANTHROPIC_API_KEY` | مفتاح API من console.anthropic.com |

### 3. رفع ملفات البيانات

```bash
# ملف متجرك (تصدير صلة/سلة)
cp "منتج_جديد.csv" input/store/

# ملفات المنافسين
cp "متجر_سارة_ميكاب.csv" input/competitors/
cp "متجر_عالم_جيفنشي.csv" input/competitors/

git add input/
git commit -m "تحديث بيانات المتجر والمنافسين"
git push
```

> سيبدأ المحرك تلقائياً عند الـ push!

### 4. تشغيل يدوي

اذهب إلى: **Actions → ✦ محرك مهووس → Run workflow**

الخيارات المتاحة:
- **توليد الأوصاف**: true/false
- **حد المنتجات**: 0 = بلا حد، أو رقم للاختبار
- **استئناف الأوصاف فقط**: لاستئناف من نقطة التوقف

### 5. تحميل النتائج

بعد انتهاء التشغيل: **Actions → Run → Artifacts → mahwous-results-N**

## ⚙️ خوارزمية المطابقة 5D

يستخدم المحرك **5 أبعاد** لمقارنة كل منتج:

| البعد | الوصف | مثال |
|:---|:---|:---|
| **الاسم النقي** | Token Set Ratio بعد تنظيف الاسم | "تشانيل ناميل" ↔ "شانيل N°5" |
| **الحجم** | المل المستخرج من الاسم | 30مل ≠ 100مل |
| **التركيز** | EDP/EDT/Parfum/Extrait | EDP ≠ EDT |
| **النوع** | عطر/تستر/طقم/معطر جسم | تستر ≠ عطر تجاري |
| **المصدر** | تتبع الملف الأصلي | متجر سارة / متجر جيفنشي |

**معايير الحكم:**
- `≥ 88%` تشابه + جميع الأبعاد متطابقة → **مكرر** (يُتجاهل)
- `≥ 88%` تشابه + اختلاف في بُعد واحد → **فرصة جديدة** 🌟
- `60–87%` تشابه → **مراجعة يدوية** 🔍
- `< 60%` تشابه → **منتج جديد تماماً** 🌟

## 🔧 متغيرات البيئة

| المتغير | القيمة الافتراضية | الوصف |
|:---|:---|:---|
| `ANTHROPIC_API_KEY` | — | **مطلوب** لتوليد الأوصاف |
| `USE_LLM` | `true` | تفعيل/تعطيل توليد الأوصاف |

| `DESCRIBE_ONLY` | `false` | استئناف الأوصاف فقط |
